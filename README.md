
**Discord News Bot** — это Discord бот для публикации новостей на сервере.
Позволяет создавать красивые сообщения с картинками, ссылками и цветами.
Имеет систему прав доступа и предпросмотра перед публикацией.

---

##  Установка и запуск

### Требования
```
Python 3.9+
disnake библиотека
Discord токен бота
```

### Запуск бота

**Способ 1 — Через .bat файл (Windows)**
```powershell
# Обычный запуск
start.bat

# Запуск с режимом отладки
startTest.bat
```

**Способ 2 — Через PowerShell/CMD**
```powershell
# Обычный запуск
py bot.py

# Запуск с режимом отладки (debug mode)
py bot.py --test
```

## 📋 Команды бота

### `/auth_news` — Регистрация сервера

**Что делает:**
```
Регистрирует сервер в системе бота
Устанавливает первую роль редактора новостей
```

**Синтаксис:**
```
/auth_news роль:<роль>
```

**Параметры:**
| Параметр | Тип | Обязателен | Описание |
|----------|-----|-----------|----------|
| `роль` | Role | ✅ Да | Роль для работы с новостями |

**Пример использования:**
```
/auth_news роль:@Редакторы
```

**Права доступа:**
```
Владелец сервера
Администраторы
Остальные — ЗАПРЕЩЕНО
```

**Что происходит:**
```
✓ Сервер получает статус "Зарегистрирован"
✓ Роль добавляется в систему прав
✓ Показывает уведомление о регистрации с временем
```

### `/send_news` — Публикация новости

**Что делает:**
```
Создаёт и публикует новость в выбранный канал
Поддерживает картинки, ссылки, цвета и упоминание ролей
```

**Синтаксис:**
```
/send_news канал:<канал> [роль:<роль>] [картинка:<файл>] [ссылка:<url>] [цвет:<hex>]
```

**Параметры:**
| Параметр | Тип | Обязателен | Описание |
|----------|-----|-----------|----------|
| `канал` | TextChannel | ✅ Да | Канал для публикации |
| `роль` | Role | ❌ Нет | Роль для упоминания |
| `картинка` | Attachment | ❌ Нет | Загруженная картинка |
| `ссылка` | Text | ❌ Нет | Ссылка на картинку (URL) |
| `цвет` | Text | ❌ Нет | Цвет в формате #HEX (например: #FF5733) |

**Примеры использования:**

**Базовая новость без картинки:**
```
/send_news канал:#новости
```

**Новость с картинкой и упоминанием роли:**
```
/send_news канал:#новости роль:@Важное картинка:[выбрать файл]
```

**Новость со ссылкой на картинку и цветом:**
```
/send_news канал:#новости ссылка:https://example.com/image.jpg цвет:#00FF00
```

**Полный вариант:**
```
/send_news канал:#новости роль:@Редакция картинка:[файл] цвет:#FF0000
```

**Права доступа:**
```
Владелец сервера
Администраторы
Пользователи с доверенной ролью (из cfg_addrole)
Остальные — ЗАПРЕЩЕНО
```

**Форма для заполнения:**
```
После команды откроется модальное окно с полями:
┌─────────────────────────────┐
│ Заголовок новости           │
│ (короткий текст)            │
├─────────────────────────────┤
│ Содержимое новости          │
│ (длинный текст)             │
└─────────────────────────────┘
```

**Поддерживаемые варианты:**
```
1. Без упоминания, без цвета
2. Без упоминания, с цветом
3. С упоминанием, без цвета
4. С упоминанием, с цветом

Картинка может быть:
- Загруженный файл
- Ссылка на картинку
- Отсутствовать вообще
```

|----------|-----|-----------|----------|
| `картинка` | Attachment | ❌ Нет | Загруженная картинка |
| `ссылка` | Text | ❌ Нет | Ссылка на картинку (URL) |
| `цвет` | Text | ❌ Нет | Цвет в формате #HEX |

**Пример использования:**
```
/preview_news ссылка:https://example.com/image.jpg цвет:#FF5733
```

**Права доступа:**
```
Владелец сервера
Администраторы
Пользователи с доверенной ролью
Остальные — ЗАПРЕЩЕНО
```

**Что происходит:**
```
✓ Форма для ввода заголовка и содержимого
✓ Показывает превью только вам
```

### `/cfg_addrole` — Добавить роль редактора

**Что делает:**
```
Добавляет роль в список доверенных
Пользователи с этой ролью смогут публиковать новости
```

**Синтаксис:**
```
/cfg_addrole роль:<роль>
```

**Параметры:**
| Параметр | Тип | Обязателен | Описание |
|----------|-----|-----------|----------|
| `роль` | Role | ✅ Да | Роль для добавления |

**Пример использования:**
```
/cfg_addrole роль:@Новые редакторы
```

**Права доступа:**
```
Владелец сервера
Администраторы
Пользователи с доверенной ролью
Остальные — ЗАПРЕЩЕНО
```

**Ответ бота:**
```
✅ [🟩] | Роль успешно добавлена в конфиг
- Роль: @Новые редакторы
- Кто добавил: @ВашеИмя
- Время добавления: [дата и время]
```

### `/cfg_reset` — Сброс конфигурации

**Что делает:**
```
Полностью сбрасывает конфиг сервера
Удаляет все роли и отменяет регистрацию
ВНИМАНИЕ: Операция необратима!
```

**Синтаксис:**
```
/cfg_reset
```

**Параметры:** Нет

**Пример использования:**
```
/cfg_reset
```

**Права доступа:**
```
Владелец сервера
Администраторы
Остальные — ЗАПРЕЩЕНО
```

**Что происходит:**
```
1. Бот выводит предупреждение
2. Появляются 2 кнопки: "Сбросить" и "Не сбрасывать"
3. После подтверждения:
   ✓ Все роли удаляются
   ✓ Сервер теряет регистрацию
   ✓ Нужно заново выполнить /auth_news
```

## 🔐 Система прав доступа

### Уровни доступа

```
Уровень 1 (Максимум) — Владелец сервера
├─ Может выполнять ВСЕ команды
└─ Всегда имеет доступ

Уровень 2 — Администраторы сервера
├─ Могут выполнять все команды
└─ Требуется права администратора

Уровень 3 — Доверенные роли
├─ Могут публиковать новости
├─ Могут управлять новостями
└─ Добавляются через /cfg_addrole

Уровень 4 (Минимум) — Обычные пользователи
├─ НЕ могут публиковать новости
└─ НЕ могут менять конфиг
```

### Таблица прав по командам

| Команда | Владелец | Админ | Доверенная роль | Остальные |
|---------|----------|-------|-----------------|-----------|
| `/auth_news` | ✅ | ✅ | ❌ | ❌ |
| `/send_news` | ✅ | ✅ | ✅ | ❌ |
| `/preview_news` | ✅ | ✅ | ✅ | ❌ |
| `/cfg_addrole` | ✅ | ✅ | ✅ | ❌ |
| `/cfg_dellrole` | ✅ | ✅ | ✅ | ❌ |
| `/cfg_info` | ✅ | ✅ | ✅ | ❌ |
| `/cfg_reset` | ✅ | ✅ | ❌ | ❌ |

## ⚠️ Обработка ошибок

| Ошибка | Причина | Решение |
|--------|--------|---------|
| `[🟥] \| Сервер не зарегистрирован` | Не выполнена `/auth_news` | Выполнить `/auth_news` |
| `[🟥] \| У вас нет прав!` | Нет нужного уровня доступа | Попросить права у владельца |
| `[🟥] \| Роль уже есть в конфиге` | Роль уже добавлена | Использовать другую роль |
| `[🟥] \| Роли нет в конфиге` | Роль не в системе | Добавить через `/cfg_addrole` |
| `[🟥] \| Неверный HEX формат` | Цвет в неправильном формате | Использовать формат `#FF5733` |
